# analysis — サーブ解析とフィードバック生成

3D関節データからバイオメカ指標を算出し、改善点を言葉で返す。
**純 numpy のみに依存**。3D復元 (GVHMR) の環境は不要で、通常の Python で動く。

## 構成

変更頻度の違う3つの層に分けている。

| モジュール | 役割 | 変更頻度 |
|---|---|---|
| `serve.py` | 計測（幾何・運動学・指標） | 低。物理計算は変わらない |
| `feedback.py` | 判定ルールと閾値 | **高**。データが貯まるたび調整する |
| `report.py` | 表示 | 将来 Web UI に置換される |

P1のパイプラインは `serve.py` だけを import すればよく、閾値の調整が
物理計算に影響しない。

## 使い方

```python
import analysis

metrics, feedback = analysis.analyze_from_files(
    "gv_joints.npy", "gv_com.npy", "gv_upaxis.npy", fps=120.0)
print(analysis.format_report(metrics, feedback))
```

CLI:

```bash
python -m analysis --joints gv_joints.npy --com gv_com.npy \
                   --upaxis gv_upaxis.npy --fps 120
```

Colab で使う場合は、このディレクトリを Google ドライブに置いて
`sys.path` に追加する（`notebooks/README.md` 参照）。

## 入力

`notebooks/` のパイプラインが出力する3ファイル。

| ファイル | 形状 | 内容 |
|---|---|---|
| `gv_joints.npy` | (F, 24, 3) | SMPL 24関節の world座標 [m] |
| `gv_com.npy` | (F, 3) | 全身重心の world座標 [m] |
| `gv_upaxis.npy` | (2,) | 上方向の軸index と符号 |

## 設計方針: プロ比較をしない

「プロと比べて差が大きい所を指摘する」は破綻する。プロ同士でもフォームは大きく異なり、
差は必ずしも欠点ではない。体格が違えば同じ関節角度は再現できないし、する必要もない。

代わりに**誰にでも当てはまる力学的原理**で判定する。参照動画も権利処理も不要で、
かつ最も価値の高い指摘が得られる。

| 判定 | 根拠 |
|---|---|
| キネティックチェーンの順序 | 体の中心から末端へ順に加速するのが力学的に効率的 |
| 打点タイミング | 重心が落ち始めてから打つと力が逃げる |
| 沈み込み・打点の高さ・捻転 | 量的指標（閾値は暫定） |

## フィードバックの作法

- **一度に1〜2件しか出さない**。人は複数の修正キューを同時に処理できず、
  15件の指摘は0件と同じになる
- 優先度は「原理違反 > 大きな逸脱 > 小さな逸脱」
- 判定はルールで決定論的に行う。将来 LLM を使う場合も**文言の生成のみ**に限定する。
  判定を任せると数値を誤解して嘘の助言を作る

## 測定できないものは主張しない

キネティックチェーンの連鎖は隣接体節の時間差が 20〜40ms しかない。
30fps では 1フレーム=33ms あり**分解できない**ため、`CHAIN_MIN_FPS`(60fps) 未満では
順序を判定せず、レポートにその旨を明示する。

実際、この保護が無かった頃は 30fps の実データで「1フレーム差」を根拠に
誤った指摘を出していた。順序を評価したい場合は 120/240fps で撮影すること。

## テスト

```bash
uv pip install -e ".[dev]"
pytest
```

`tests/synth.py` が合成サーブを生成する。タイミングは秒で定義してあり、
fps を変えても同じ実時間の動作になるため、フレームレート依存の挙動を検証できる。

このテストは実際に2つのバグを検出した。閾値やルールを触ったら必ず走らせること。

## 既知の限界

- **閾値が暫定**。順序・タイミング系は理論から断定できるが、
  「膝を何度まで曲げるべきか」等は文献値かデータ蓄積で確定させる必要がある
- **コート座標が未定義**。前後左右が分からないため踏み込み方向を評価できない
- **1本のサーブのみ**を前提。複数本の比較・経時変化の追跡は未実装
