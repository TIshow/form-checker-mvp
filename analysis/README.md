# サーブ解析・フィードバック生成

`serve_analysis.py` — 3D関節データからバイオメカ指標を算出し、改善点を言葉で返す。

**純 numpy のみに依存**。GVHMR環境(Python 3.10 venv)は不要で、Colabの通常カーネルでも
ローカルでも動く。入力は `notebooks/` のパイプラインが出力する `.npy` 3点のみ。

## 設計方針: プロ比較をしない

「プロと比べて差が大きい所を指摘する」は破綻する。プロ同士でもフォームは大きく異なり、
差は必ずしも欠点ではないし、体格が違えば真似できない。

代わりに**誰にでも当てはまる力学的原理**で判定する:

| 判定 | 根拠 | プロ動画 |
|---|---|---|
| キネティックチェーンの順序 | 体の中心から末端へ順に加速するのが力学的に効率的 | 不要 |
| 打点タイミング | 重心が落ち始めてから打つと力が逃げる | 不要 |
| 沈み込み・打点の高さ・捻転 | 量的指標（しきい値は暫定） | 不要 |

## Colab での使い方

初回のみ、このファイルをGoogleドライブに置く（`MyDrive/gvhmr_backup/serve_analysis.py`）。

```python
import sys
sys.path.insert(0, '/content/drive/MyDrive/gvhmr_backup')
import serve_analysis as sa

metrics, feedback = sa.analyze_from_files(
    '/content/gv_joints.npy',
    '/content/gv_com.npy',
    '/content/gv_upaxis.npy',
    fps=30.0,
)
print(sa.format_report(metrics, feedback))
```


## 出力される指標

- **フェーズ**: 沈み込み / 重心の頂点 / 打点 のフレーム
- **脚のドライブ**: 重心の最低点・最高点・伸び上がり量と所要時間、沈み込み時の膝角度
- **打点**: 高さ（頭の高さに対する比で正規化）、重心頂点とのタイミング差、肘の角度、体幹の傾き、最大捻転差
- **キネティックチェーン**: 腰→肩→上腕→前腕 の角速度ピーク時刻

## フィードバックの原則

- **一度に1〜2件しか出さない**。人は複数の修正キューを同時に処理できず、15件の指摘は0件と同じ
- 優先度は「原理違反 > 大きな逸脱 > 小さな逸脱」
- 判定はルールで決定論的に行う。将来LLMを使う場合も、**文言の生成のみ**に限定する
  （判定をLLMに任せると数値を誤解して嘘の助言を作る）

## 既知の限界

- **しきい値が暫定**。順序・タイミング系は理論から断定できるが、
  「膝を何度まで曲げるべきか」等の量的しきい値は、データ蓄積か文献値で確定させる必要がある
- **コート座標が未定義**。前後・左右が分からないため、踏み込み方向は評価できない
- **1本のサーブのみ**を前提。複数本の比較・経時変化の追跡は未実装
- 回転が小さいセグメントは角速度ピークがノイズになるため、順序判定から自動除外している

## 検証

合成サーブデータで、良いフォーム／連鎖の逆転／遅れた打点 の3ケースを判別できることを確認済み。
