# Issue 001: フェーズ検出を「自動検出→人が微調整」にし、打点を打球音で精密化する

Status: Open
Created: 2026-07-22
GitHub: [#1](https://github.com/TIshow/form-checker-mvp/issues/1)
関連コード: [analysis/serve.py](../../analysis/serve.py) `detect_phases`

---

## 背景 / 現状の問題

現在のフェーズ検出（[analysis/serve.py](../../analysis/serve.py) `detect_phases`）は3行で、
すべて単純な最大/最小で決めている。

```python
contact  = argmax(wrist_h)              # ラケット側手首が最も高いフレーム
loading  = argmin(com_height[:contact]) # 打点より前で重心が最も低いフレーム
com_peak = argmax(com_height)           # 重心が最も高いフレーム（クリップ全体）
```

これには3つの問題がある。

1. **打点はボールを見ていない。** 「手首の最高点」を代理にしているだけ。
   実際の接触点はラケット分だけ手首より上で、フォロースルーでも手首は上がりうる。
   P0では frame 103 が実際の接球と一致したが、1本での結果にすぎない。

2. **`com_peak` の探索範囲が無制限。** クリップ全体から最大を取るため、
   サーブ後のジャンプや歩行を「重心の頂点」として拾いうる。これが起きると
   打点タイミングの判定（Tier A、断定してよいはずの指摘）が壊れる。

3. **`loading` の前側に下限が無い。** 打点より前ならどこでもよいので、
   ボールを拾うためにしゃがんだ動作を「沈み込み」と誤検出しうる。

つまり現状は「**1クリップにサーブ1本だけが写っている**」という暗黙の前提に
依存している。実際の練習動画は複数動作が混在するため、この前提は成り立たない。

## なぜ重要か

Tier A（断定してよい）の指摘のほぼ全てが**打点フレームを基準に計算**される。

- 打点タイミング（重心の頂点との差）
- 接球時の肘の角度
- キネティックチェーンの加速区間の終点

打点がずれると、これらが連鎖的にずれる。**打点はシステムで最も脆く、最も重要な点。**

---

## 方針

### 1. 自動検出 → 人が微調整

「毎回2箇所を手入力する必須ステップ」にはしない。それは
「練習動画を撮ったら自動でフィードバック」というプロダクトのビジョンと衝突する。

```
悪い設計: 開始と打点を手で入れないと動かない
良い設計: 自動で推定 → 「ここでいい?」と提示 → ズレていたら人が1コマ動かす
```

自動が当たれば速く、外れても人が直せる。将来の完全自動化への道も塞がない。

### 2. 打点を打球音で精密化する（映像で絞った窓の中で）

音を動画全体から探すと、他コートの打球音・声・風を誤検出する。そこで：

```
① 映像で打点をおおまかに推定（手首の最高点）
② その周辺 ±0.3秒 の音声だけを見る
③ 窓の中で最も鋭い衝撃音のピークを打点とする
```

映像が「だいたいこの辺」を教え、音が「正確にこの瞬間」を返す。
2つのセンサーが弱点を補い合う。

打球音（ラケットの「パコッ」）は短く・高周波を多く含む・急に立ち上がる過渡音。

1. 動画から音声を抽出（モノラル）
2. バンドパス（およそ1〜8kHz）で低域のうなりと声を落とす
3. 包絡線 or オンセット強度（立ち上がりの鋭さ）を計算
4. ①の窓の中でそのピークを取る → その時刻が打点

音声は44.1kHzなのでサブミリ秒精度。30fps映像の33ms粒度より正確で、
**人が目視で指すより正確**。最後に `frame = round(t_hit × fps)` で映像に対応づける。

### 3. サーブ区間の境界（開始点）

体重移動の**計算精度**は開始点に依存しない（各フレーム独立に重心を出しており、
誤差が蓄積しない）。開始点が要るのは**区間の切り出し＝頑健性**のため。
特に `loading` 探索の前側の下限を与え、無関係な動作の混入を防ぐ。

自動検出の初期実装としては「打点から一定時間さかのぼった範囲」を既定とし、
人が調整できるようにする。

## 受け入れ条件

- サーブ以外の動作を含むクリップでも、打点・沈み込みが正しい区間から選ばれる
- 打点・沈み込みは自動検出され、かつ人の値で上書きできる
- 打球音が使える録音では、音由来の打点が映像推定より安定する
- 音が使えない録音でも、手動上書きで正しい打点を与えられる
- 上記が合成データのテストで保証される

## リスク / 未解決

- **この録音の音質は未知。** フェンス越し・背後に別プレイヤーで、窓の±0.3秒内に
  別の衝撃音が入ると誤りうる。フェーズ A で実データを見て判断する
- **30fps では手動でも打点は±1〜2コマ(33〜66ms)ずれる。** タイミング閾値0.10秒に
  対して無視できない。120fps 撮影が根本的な改善策
- **音声と映像の同期ずれ。** 通常のスマホ録画では問題ないが、確認は必要
- 開始点の既定さかのぼり時間は暫定。データを見て調整する

- 複数サーブの一括切り出し・自動カウント
