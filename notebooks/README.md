## このノートの肝

### Python 3.10 環境が必須
Colab標準の **Python 3.12 では動きません**。`numpy==1.23.5` がビルド不可、
`chumpy` が py3.11+ で死ぬ、`pytorch3d` が cp310 専用 wheel のため。
→ `uv` で Python 3.10 の venv を作り、そこで GVHMR を実行する（セクション3）。
ノートのカーネルは 3.12 のままで、解析結果は `.npy` 経由で受け渡す。

### 重心の計算方法
GVHMR の `smpl_params_global` は名前に反して **SMPL-X パラメータ**（`body_pose` が63）。
GVHMR本体と同じ経路をたどる:

```
SMPL-X で推論 → smplx2smpl で SMPL頂点へ変換 → J_regressor で24関節
→ De Leva の体節質量比で全身重心
```

これにより **メートル単位の物理的に正しい重心**が得られる。
（旧MVPの「2Dピクセル加重平均」は単位も基準も無く物理的に無意味だった）

## 実測結果の例（2026-07-19, temp_my_serve.mp4）

| 指標 | 値 |
|---|---|
| 重心の高さ範囲 | 0.890 – 1.120 m |
| 沈み込み（トロフィーポーズ） | frame 87 / 0.953 m |
| **打点（実際の接球と一致）** | **frame 103 / 1.120 m** |
| 伸び上がり | +0.167 m / 0.53 秒 |
| 水平移動 | 1.174 m |

重心の絶対高さが身長の約55%という解剖学的に妥当な値に収まり、
**重心の最高点が動画上の実際の打点と一致**した。これが P0 の合格判定。

## 次の課題

1. **フェーズ自動分割** — 動画にサーブ以外の動作が混在するため、サーブ区間の切り出しが必要
2. **人物追跡の頑健化** — 背景に別プレイヤーがいると追跡が移る可能性
3. **指標の拡充** — 肘の角度、肩の回転、キネティックチェーンなど
4. **コート座標の定義** — 現状 axis 0 / axis 2 がコートのどちら向きか未定義のため水平移動を解釈できない
